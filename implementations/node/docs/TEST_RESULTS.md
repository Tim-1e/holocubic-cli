## Issues found by platform/hardware testing

1. ESP32 SD writes can exceed a short desktop HTTP timeout. The default was
   increased to 60 seconds; E2E allows 120 seconds.
2. The Lua JSON encoder represents an empty table as `{}`. The client accepts
   only a keyless object as an empty list and still rejects other malformed
   list responses.
3. npm POSIX commands use a symlink. CLI entry detection now compares real
   paths so the installed Linux command actually executes.
4. Simulated Windows/Linux config paths now use `path.win32` and `path.posix`
   explicitly, preventing host-platform-dependent tests.
